## MQTT 后端对接文档（阈值设置相关）

### 1. 上行温湿度上报格式

设备定时向上行主题上报当前温湿度，payload JSON 结构如下：

```json
{
  "device_id": "<设备唯一ID>",
  "timestamp": 1710000000000,
  "temp": 23.45,
  "humi": 56.78
}
```

- **device_id**：设备唯一 ID，由设备名 + MAC + IMEI 组成，例如：  
  `esp32_ml307_client_98:A3:16:F9:F5:10_866522075961033`
- **timestamp**：Unix 毫秒时间戳，优先使用网络时间，若未同步则为自上电以来的毫秒数。
- **temp**：当前温度，单位 ℃。
- **humi**：当前湿度，单位 %RH。

### 2. 下行报警阈值设置指令（云 → 设备）

后端向指令主题（示例：`cmd/esp32_ml307_client`）下发 JSON，用于远程设置设备的温度 / 湿度报警阈值。

#### 2.1 通用规则

- 设备会优先检查 payload 中的 `device_id` 字段：
  - 如果存在且 **与本设备的 device_id 不一致**，指令会被**忽略**。
  - 如果不存在，则默认认为是“广播指令”，当前实现会**不校验** device_id（建议实际使用时都带上 device_id）。
- 阈值字段：
  - **temp_th**：温度报警阈值（℃），范围限制在 `[-40, 125]`。
  - **humi_th**：湿度报警阈值（%RH），范围限制在 `[0, 100]`。
  - 两个字段都可选，可只下发其中一个。

#### 2.2 典型示例

- **同时设置温度 30.5℃、湿度 65% 阈值：**

```json
{
  "device_id": "esp32_ml307_client_98:A3:16:F9:F5:10_866522075961033",
  "temp_th": 30.5,
  "humi_th": 65
}
```

- **只修改温度阈值（示例：28℃）：**

```json
{
  "device_id": "esp32_ml307_client_98:A3:16:F9:F5:10_866522075961033",
  "temp_th": 28
}
```

- **只修改湿度阈值（示例：55%）：**

```json
{
  "device_id": "esp32_ml307_client_98:A3:16:F9:F5:10_866522075961033",
  "humi_th": 55
}
```

#### 2.3 设备端处理逻辑摘要

- 收到指令 → 解析 `device_id`：
  - 若存在且不等于本机 `device_id`：忽略本条指令。
  - 若相等或字段不存在：继续解析阈值。
- 解析 `temp_th` / `humi_th`，做范围裁剪后写入全局变量：
  - `g_temp_threshold`
  - `g_humidity_threshold`
- 修改后的阈值会自动影响：
  - LCD 显示的 `T_Th` / `RH_Th` 行；
  - 报警任务（红灯 / 黄灯 / 蜂鸣器）的触发条件。

### 3. 上行报警状态消息（设备 → 云）

当设备检测到“是否报警”的状态发生变化时，会额外上报一条报警状态消息，使用与温湿度上报相同的 topic。

#### 3.1 报文结构

```json
{
  "device_id": "<设备唯一ID>",
  "timestamp": 1710000000000,
  "temp": 30.50,
  "humi": 80.00,
  "temp_alarm": true,
  "humi_alarm": false
}
```

- **device_id**：同上。
- **timestamp**：同上。
- **temp/humi**：当次判断报警时刻的温度 / 湿度。
- **temp_alarm**：布尔值，`true` 表示温度超过当前温度阈值 `g_temp_threshold`。
- **humi_alarm**：布尔值，`true` 表示湿度超过当前湿度阈值 `g_humidity_threshold`。

#### 3.2 触发规则

- 设备周期读取温湿度（约每 5 秒一次），并根据当前阈值判断是否报警。
- **仅当报警状态发生变化时**会上报报警消息一次：
  - 例如从“无报警 → 有温度报警”；或从“有温度报警 → 无报警”；或“只有温度报警 → 温度+湿度同时报警”等。
- 报警状态不变时**不会重复上报**，避免 MQTT 消息过多。
